# java.util.List
## 一句话总结

java.util.List 是一个有序、可重复的集合接口，其常用实现包括：

ArrayList：基于动态数组，随机访问快，增删慢（除非在末尾）。

LinkedList：基于双向链表，增删快，随机访问慢。

Vector：线程安全的动态数组（已过时，被 ArrayList 与 Collections.synchronizedList 取代）。

CopyOnWriteArrayList（位于 java.util.concurrent 包）：线程安全的List，采用写时复制技术，读操作无锁且极快，适用于读多写极少的高并发场景。

最终总结：在单线程环境下，ArrayList 是最通用的选择；在高并发且读多写极少的场景下，CopyOnWriteArrayList 是重要的线程安全解决方案。Vector因性能原因已不再推荐用于新代码。



## 深入了解

### 什么是动态数组？

**动态数组**是一种数据结构，它能够在运行时根据需要自动调整其容量。在Java中，`ArrayList`就是基于动态数组实现的。

**核心原理**：

- 动态数组在内部使用一个固定大小的数组来存储元素。当数组被填满时，它会执行一个“扩容”操作：创建一个新的、更大的数组（通常是原数组的1.5倍，在Java的`ArrayList`中，扩容因子是1.5），然后将旧数组中的所有元素复制到新数组中，最后将新数组作为存储容器。

- 这种自动扩容的特性使得动态数组能够提供类似数组的随机访问性能（O(1)），同时又能灵活地处理元素数量的变化。

**优点**：

- 支持快速随机访问（通过索引直接访问）。
- 内存连续，缓存友好，遍历效率高。

**缺点**：

- 扩容操作代价较高，涉及旧数组的复制。

- 在数组中间插入或删除元素需要移动后续所有元素，效率较低。

  

### ArrayList 为什么增删慢，而在末尾快？

**ArrayList 的增删操作效率分析**：

- **在末尾添加元素**：
  - 如果数组容量足够，只需要在数组末尾放入新元素，然后更新size，时间复杂度为 **O(1)**。
  - 如果数组已满，则需要先扩容（创建一个新数组并复制元素），然后再添加，这次操作的时间复杂度为O(n)，但均摊下来仍然是O(1)（因为扩容不是每次添加都发生）。
- **在中间或开头插入/删除元素**：
  - 插入时，需要将插入点之后的所有元素向后移动一位，为新的元素腾出空间。
  - 删除时，需要将删除点之后的所有元素向前移动一位，填补空缺。
  - 移动元素的时间复杂度为 **O(n)**，因为平均可能需要移动一半的元素。

**结论**：在末尾操作快（O(1)），而在中间或开头操作慢（O(n)），是因为数组的连续内存特性导致元素必须紧密排列，任何非末尾的插入删除都会引发大量元素的移动。



### 什么是双向链表？

**双向链表**是一种线性数据结构，其中每个节点（Node）包含三部分：

- 数据域（存储元素）
- 前驱指针（指向前一个节点）
- 后继指针（指向后一个节点）

在Java的`LinkedList`中，它就是一个双向链表的实现。

**特点**：

- 节点在内存中不是连续存储的，而是通过指针链接在一起。
- 可以从任意一端（头或尾）开始遍历链表。
- 插入和删除节点时，只需要改变相邻节点的指针指向，不需要移动其他节点。

**优点**：

- 插入和删除操作效率高（只需改变指针）。
- 不需要预先分配固定空间，可以动态增长。

**缺点**：

- 不支持随机访问，必须从头部或尾部开始遍历，直到找到目标节点。
- 每个节点需要额外的空间存储前后指针，内存开销较大。
- 由于内存不连续，对CPU缓存不友好，遍历速度可能慢于数组。



### LinkedList 为什么增删快，随机访问慢？

**LinkedList 的增删操作**：

- 在链表已知位置（如头、尾或给定节点）进行插入或删除，只需要修改相邻节点的指针，时间复杂度为 **O(1)**。
- 但是，如果要在指定索引位置插入或删除，需要先遍历到该位置（O(n)），然后修改指针（O(1)）。因此，按索引增删的总体时间复杂度为O(n)。不过，`LinkedList`在头部和尾部的操作是O(1)，因为其内部维护了头尾指针。

**LinkedList 的随机访问**：

- 链表不支持像数组那样通过索引直接计算内存地址。要访问第i个元素，必须从头节点（或尾节点）开始，逐个向后（或向前）遍历，直到第i个节点。这个过程的时间复杂度为 **O(n)**。

**对比**：

- 增删快是相对于数组的“移动元素”而言，链表只需修改指针，但前提是已经定位到要操作的节点。如果要从头开始查找位置，则增删的总时间也是O(n)。
- 随机访问慢是因为链表无法通过索引直接寻址，必须顺序遍历。

**实际使用注意**：

- 在`LinkedList`中，`add(E e)`方法在末尾添加是O(1)，因为直接操作尾指针。
- `add(int index, E element)`方法需要先遍历到index位置，所以是O(n)。
- 同理，`get(int index)`也是O(n)。



# java.util.Map

## 一句话总结

java.util.Map 是一个存储键值对的接口，其常用实现包括：

- 非线程安全：HashMap（无序哈希表，最常用）、LinkedHashMap（保持插入或访问顺序）、TreeMap（基于红黑树的有序映射）。
- 线程安全：
  - Hashtable：早期实现，全表锁，性能差，已过时。
  - ConcurrentHashMap：分段锁/CAS优化，高并发首选，性能极高。
  - ConcurrentSkipListMap：基于跳表的有序并发映射。
- 核心对比：单线程用HashMap，高并发用ConcurrentHashMap，需要有序并发则用ConcurrentSkipListMap。Hashtable和synchronizedMap（包装类）因性能问题不推荐用于新系统。
